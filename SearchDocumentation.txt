Search Documentation

Overview
The current console application supports three top-level search paths through QueryMenu:
- Cards
- Games
- Sets

Card and set searches collect user input from dedicated service classes, convert those values into query parameter models, translate the populated models into dictionaries, URL-encode the values, and append the final query string to the correct JustTCG endpoint. Game searches call the /games endpoint directly with no additional parameters.

Implemented Search Flow
1. Program initializes the shared HTTP client header through FetchApi.SetApiHeader().
2. QueryMenu.Run() prompts the user to choose Card, Game, or Set.
3. InputJustTCGCardQuery.Run() or InputJustTCGSetQuery.Run() gathers validated console input when the selected endpoint requires parameters.
4. ObjToDict.Run() converts non-null query model properties into lowercase key/value pairs.
5. BuildQuery.Run() URL-encodes the values and produces the final query string.
6. FetchApi.Run() sends the authenticated GET request to https://api.justtcg.com/v1.
7. DeserializeApi.Run<T>() maps the JSON response into ApiResponse<T> so the application can work with strongly typed models.

Classes and Responsibilities
- InputJustTCGCardQuery
  Collects card search criteria from the console and returns a JustTCGCardQueryParams object.
- InputJustTCGSetQuery
  Collects set search criteria from the console and returns a JustTCGSetQueryParams object.
- QueryMenu
  Selects the endpoint workflow, builds the request path, executes the API call, and chooses the matching response model.
- BuildQuery
  Builds a URL-safe query string from a dictionary of parameters.
- ObjToDict
  Converts populated query model properties into a dictionary for request building.
- FetchApi
  Applies the JustTCG API key header and performs HTTP GET requests.
- DeserializeApi
  Deserializes raw JSON into ApiResponse<T>.

Query Parameter Models
JustTCGCardQueryParams
- Q
  Required free-text search term for the card lookup.
- Number
  Optional card number filter.
- Printing
  Optional printing filter, such as Normal or Foil.
- Condition
  Optional condition filter using the Condition enum values: S, NM, LP, MP, HP, DMG.

JustTCGSetQueryParams
- Q
  Optional free-text set search term.
- Game
  Required game filter, such as mtg or pokemon.
- OrderBy
  Optional sort field using the OrderBy enum values: name or release_date.
- Order
  Optional sort direction using the Order enum values: asc or desc.

Supporting Models
- ApiResponse<T>
  Wraps the API data payload, paging metadata, usage metadata, and error fields.
- JustTCGCard, JustTCGSet, JustTCGGame, Variant, Meta, and Metadata
  Represent the JSON structures returned by the JustTCG endpoints.

Additional Search Client
Search.cs also includes a typed JustTcgCardsClient prototype. That client normalizes CardSearchQuery values, builds a fully qualified cards URI, authenticates with the JUSTTCG_API_KEY environment variable, and maps the raw API response into simplified CardSearchResult records with paging metadata.

Authentication
The project expects the API key to be supplied through the JUSTTCG_API_KEY environment variable. No API credentials should be stored in source files.
